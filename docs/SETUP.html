Prerequisites

You need Python 3.10+ and pip. Confirm with:

terminal
python --version
pip --version

If targeting Supabase, have your project URL and service-role key ready from the Supabase dashboard (Settings → API).

1

Navigate to the backend directory

The backend lives at CalculusRuntime/backend/ inside the repo.

terminal
cd backend
2

Create a virtual environment

bash
python -m venv .venv
source .venv/bin/activate
PowerShell
python -m venv .venv
.\.venv\Scripts\Activate.ps1

Your prompt should now show (.venv) as a prefix.

⚠ On Windows, if execution policy blocks the activation script, run Set-ExecutionPolicy -Scope CurrentUser RemoteSigned first.
3

Install dependencies

terminal
pip install -r requirements.txt

4

Configure environment variables

Copy the example file and fill in your values:

bash
cp .env.example .env
PowerShell
Copy-Item .env.example .env
Auth & tokens
SECRET_KEY JWT signing secret. Replace before any non-local use. Default is a placeholder.
TOKEN_EXPIRE_MINUTES JWT lifetime. Default: 10080 (7 days).
Database (SQLite)
DB_PATH Path to the SQLite file. Default: calcvoyager.db.
CORS
ALLOWED_ORIGINS Comma-separated list of allowed origins. Default: http://localhost:3000,http://127.0.0.1:3000
Database (Supabase)
SUPABASE_URL Your Supabase project URL.
SUPABASE_SERVICE_ROLE_KEY Backend-only service role key. Never expose to the client.
DB_BACKEND Set to supabase to activate. Any of DB_BACKEND, DATABASE_BACKEND, or PROGRESS_DB work.
⚠ The .env file is git-ignored. Never commit it. Never commit calcvoyager.db either.
5

Set up the database

SQLite (default — no extra steps)

On first startup db.py runs CREATE TABLE IF NOT EXISTS for all five tables automatically. No manual migration needed. The database file appears at the path set by DB_PATH (default: calcvoyager.db).

✓ Tables created: users, sections, bookmarks, quiz_scores, solver_history. Foreign keys cascade deletes from users.

Supabase (optional)

The schema is not applied automatically for Supabase. Run it once in the Supabase SQL Editor before starting the backend:

Supabase → SQL Editor → New query
-- paste the contents of backend/supabase_schema.sql and run it

The schema creates the same five tables with bigint identity PKs, adds indexes on user_id columns, and disables RLS on all app-owned tables (the backend uses its own JWT and writes with a service-role key, not Supabase Auth).

⚠ Always use SUPABASE_SERVICE_ROLE_KEY — not the anon/publishable key — or Supabase will block backend writes with a 403.
6

Start the server

development (hot reload)
uvicorn main:app --reload --port 8002

The server starts at http://127.0.0.1:8002. The --reload flag restarts on file changes.

ℹ SQLite uses WAL mode and is not safe for multiple concurrent worker processes. Run a single worker locally.
7

Verify the installation

Check the health endpoint:

terminal
curl http://127.0.0.1:8002/api/health
expected response
{"status": "ok"}

Run a full signup + protected route check:

bash — signup
TOKEN=$(curl -s -X POST http://127.0.0.1:8002/api/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"username":"testuser","password":"testpass1"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")
echo "Token: $TOKEN"
bash — fetch profile
curl http://127.0.0.1:8002/api/auth/me \
  -H "Authorization: Bearer $TOKEN"
PowerShell — signup
$resp = Invoke-RestMethod -Method POST `
  -Uri "http://127.0.0.1:8002/api/auth/signup" `
  -ContentType "application/json" `
  -Body '{"username":"testuser","password":"testpass1"}'
$token = $resp.access_token
PowerShell — fetch profile
Invoke-RestMethod -Uri "http://127.0.0.1:8002/api/auth/me" `
  -Headers @{ Authorization = "Bearer $token" }

You can also inspect the SQLite schema at any time:

terminal
sqlite3 calcvoyager.db .schema

Interactive API docs are served at http://127.0.0.1:8002/docs (custom HTML maintained in main.py — not Swagger).

Troubleshooting

401 Not authenticated
Missing or invalid bearer token.
→ Log in or sign up to get a token. Pass it as Authorization: Bearer <token>.
CORS failure in the browser
Frontend origin not in ALLOWED_ORIGINS.
→ Add the exact origin (including port) to ALLOWED_ORIGINS in .env and restart.
500 — Supabase schema cache error
Tables haven't been created in Supabase yet.
→ Run supabase_schema.sql in the Supabase SQL Editor, then restart the backend.
500 — row-level security / 42501
Using the anon/publishable key instead of the service-role key, or RLS is re-enabled.
→ Use SUPABASE_SERVICE_ROLE_KEY. Re-run the schema SQL to reset RLS settings.
SQLite database locked
Multiple Uvicorn workers or processes sharing the same .db file.
→ Run a single worker locally. For multi-process production use Supabase.
Password always rejected after a user update
Wrong DB_PATH, or the hashed_pw column was modified directly.
→ Confirm DB_PATH points to the right file. Recreate the test user via /api/auth/signup.
Syntax check before a PR
Catching import or syntax errors early.
